# Services reference

Ollie's current service offerings, and how they map to the kinds of problems a heuristic review turns up. Edit this file directly to update the list, add/remove services, or tweak the mapping — the skill re-reads it every run, no need to touch `SKILL.md` itself.

## Services

- Conversion funnel audit (checkout, signup, booking, trial)
- Usability testing (moderated / unmoderated + findings report)
- Accessibility audit (WCAG spot-check)
- Information architecture review
- Form design audit or redesign
- Microcopy / UX writing pass (errors, empty states, CTAs)
- Onboarding flow audit or redesign
- Competitive UX benchmarking (you vs. 2–3 competitors)
- Landing page conversion audit
- Card sort / IA workshop
- Design system consistency audit
- Wireframe or prototype for one specific flow fix
- Persona / user journey mapping session

## Problem → service mapping

Use this when writing the closing "how I can help" section. It's many-to-many — a single finding can point at more than one service, and a service can resolve findings from more than one focus lens. Pick whichever service(s) most directly resolve the specific finding, don't force every finding into a slot.

| Kind of problem found | Likely service(s) |
|---|---|
| Friction in checkout, signup, booking, or trial flows | Conversion funnel audit |
| General task-completion confusion, unclear next steps | Usability testing |
| Contrast, focus states, screen-reader/semantic issues, keyboard access | Accessibility audit (WCAG spot-check) |
| Confusing navigation, unclear site structure, users can't find things | Information architecture review; Card sort / IA workshop |
| Form friction — validation, labelling, error handling, field order | Form design audit or redesign |
| Jargon, unclear labels, unhelpful error messages, weak CTAs | Microcopy / UX writing pass |
| New users confused in their first session, unclear first-run experience | Onboarding flow audit or redesign |
| A landing/marketing page isn't converting visitors | Landing page conversion audit |
| Inconsistent components, spacing, or patterns across pages | Design system consistency audit |
| One specific flow is broken and needs a concrete fix, not just a diagnosis | Wireframe or prototype for one specific flow fix |
| Uncertainty about who the users actually are or how they think | Persona / user journey mapping session |
| Client wants to know how they compare to rivals | Competitive UX benchmarking |

## No direct match

Some findings (e.g. performance-adjacent observations that come up incidentally) won't map cleanly to any service above. Don't force a mapping — report the finding on its own merits and let the closing section's general "not sure which of these fits, let's talk" contact prompt cover it.
